# User Interface Design (Lecture 14)

## 1. Evolution of User Interfaces

- **Command‑line interfaces (CLI)** – text only, expert users.
- **Graphical user interfaces (GUI)** – point‑and‑click, windows, icons.
- **Web applications** – browser‑based interfaces.
- **Mobile applications** – touch‑based, gestures.

---

## 2. UI Syntax (Look & Feel) – Layout Dimensions

| Dimension | Description | Example |
|-----------|-------------|---------|
| **1D layout** | Menu bar along x‑direction only | MS Notepad |
| **2D layout** | Menu bar (x) + side/ bottom panels (y) | TextPad, Paint |
| **3D layout** | Menu (x) + toolbars (y) + tabs (z) | Eclipse, VS Studio |
| **4D layout** | 3D + changing menu sets (extra direction) | Maya (Animation, Modeling tabs) |

---

## 3. UI Styles (Based on Appearance)

- **Static** – fixed layout, no user changes.
- **Dynamic** – adapts to user actions.
- **Customizable** – user can rearrange/configure.

## 4. Command Styles (Based on Interaction)

- **Textual** (CLI) – fading out.
- **Graphical** (GUI) – dominant.
- **Other** – voice, gesture.

---

## 5. UI Semantics – Usability

| Concept | Meaning |
|---------|---------|
| **Usability** | Ease of use, learnability, efficiency, error reduction. |
| **Consistency** | Similar commands/menus behave the same. |
| **Integration** | UI works smoothly with system functions. |

> “A beautiful UI does **not** guarantee good usability.”

---

## 6. Human Factors in UI Design

| Factor | Implication |
|--------|--------------|
| **Limited short‑term memory** | ~7 items – avoid overloading. |
| **People make mistakes** | Provide undo, confirmations. |
| **People are different** | Support different skills, preferences, physical abilities. |

---

## 7. GUI Advantages & Problems

| Advantages | Problems |
|------------|----------|
| Easy to learn and use | Not automatically good |
| Switch tasks quickly | Many systems unused due to poor UI |
| Fast full‑screen interaction | Catastrophic errors possible |
| Information remains visible | |

---

## 8. UI Design Principles (from slides)

| Principle | Explanation |
|-----------|-------------|
| **User familiarity** | Use real‑world terms (e.g., “documents”, not “file identifiers”). |
| **Consistency** | Same format, same punctuation across similar commands. |
| **Minimal surprise** | Predictable behaviour – like other familiar systems. |
| **Recoverability** | Undo, confirm destructive actions, soft deletes. |
| **User guidance** | Help systems, online manuals, tooltips. |
| **User diversity** | Support different needs (larger text, colour blindness). |

---

## 9. Web Application Design – Quick Summary

- **Goal:** Usability, scalability, performance, security, maintainability.
- **Components:** Frontend (UI/UX, HTML/CSS/JS, React/Angular), Backend (business logic, Node.js/Spring/Django), Database, APIs.
- **Architectures:** Monolithic, Client‑Server.

**Security:** Authentication, encryption, secure APIs, attack protection.

---

## 10. Mobile Application Design – Quick Summary

- **UI (look)** + **UX (feel)**.
- **Goals:** Simple navigation, fast performance, consistent across devices.
- **Components:** Layout, navigation, visual design, interaction design (tap, swipe).
- **Principles:** Simplicity, consistency, accessibility, feedback, performance.
- **Approaches:** Native (Android/iOS), Responsive (adapts), Adaptive (different layouts).
- **Challenges:** Small screen, device fragmentation, battery constraints.
